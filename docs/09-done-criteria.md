# Done Criteria

A task is only done if:
- it is implemented
- it matches MVP scope
- it preserves the premium direction
- it does not break unrelated parts
- it is modular enough
- it is typed where relevant
- it keeps mocks replaceable
- it does not create future integration traps

A phase is only done if:
- all required tasks for the phase are complete
- the result feels coherent and premium
- demo flows feel believable and usable
