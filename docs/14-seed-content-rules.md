# Seed Content Rules

## Goal

The mock content should make the product feel believable and premium.

## Minimum Seed Requirements

Create at least:
- 12 to 18 realistic assets
- across the 5 initial categories
- with enough variation to make the library feel real

## Per Asset Required Fields
- id
- slug
- title
- short_description
- full_description
- category
- platform_type
- style_type
- thumbnail_url
- preview_images
- psd_file_key
- file_size_mb
- version
- is_featured
- is_published
- tags

## Preview Rules
Each asset should have:
- 1 thumbnail image
- 2 to 4 preview images

Use placeholder files or reusable mock previews if necessary, but structure them realistically.

## Quality Rule
Seed assets should feel intentional, not random filler.

The asset titles, tags, and categories should reinforce the premium product concept.
