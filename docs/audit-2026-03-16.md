### B1 ‚Äî PSD download route streams files through serverless function
**What:** `/api/download/[assetId]/route.ts` proxies the entire PSD file (10‚Äì25MB) through the Next.js serverless function instead of redirecting to the signed URL.
**Why:** On Vercel Hobby (4.5MB limit), every download returns a corrupt/truncated file. On any tier, this adds 2‚Äì5+ seconds of unnecessary latency and wastes egress bandwidth. The comment explains the motivation ("so browser's `download` attribute works") but this can be solved without proxying.
**Where:** `app/api/download/[assetId]/route.ts` lines 56‚Äì70
**How:** Return the signed URL as JSON: `return NextResponse.json({ url: data.signedUrl, filename: '${asset.slug}.psd' })`. In `components/members/download-button.tsx`, instead of `res.blob()`, do: `const { url, filename } = await res.json(); const a = document.createElement('a'); a.href = url; a.download = filename; a.click()`. Note: the `download` attribute on cross-origin URLs is ignored by browsers ‚Äî handle this by showing a "Your download has started" message or opening in a new tab for cross-origin. If the signed URL is on the same domain (via a reverse proxy or custom R2 domain), the `download` attribute will work.
