# UI Rules

## Overall Rule

The UI must feel premium, restrained, polished, and calm.

If something feels generic, cluttered, noisy, or cheap, it is wrong.

## Mandatory UI Qualities
- strong spacing
- clean typography
- clear hierarchy
- premium dark aesthetic
- subtle contrast
- attractive preview presentation
- smooth visual rhythm
- minimal clutter
- responsive layout
- consistent component styling

## Forbidden UI Qualities
Do not create:
- loud gradients everywhere
- excessive glow effects
- cluttered cards
- cheap marketplace aesthetics
- busy dashboards
- inconsistent spacing
- weak hierarchy
- overdecorated sections
- visual gimmicks that reduce clarity

## Product Shell Rule

Even in demo or mock states, the UI should feel real and polished.
Mock flows must not feel broken or unfinished.
