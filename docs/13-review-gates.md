# Review Gates

## Phase 1 Gate
Do not continue unless:
- project structure is clean
- design system exists
- temporary branding feels premium
- reusable primitives exist
- nothing depends on external services

## Phase 2 Gate
Do not continue unless:
- homepage feels premium
- pricing page feels coherent
- navigation is clean
- visuals do not feel generic or cheap

## Phase 3 Gate
Do not continue unless:
- mock login/logout flows work
- member guard works
- admin guard works
- mock auth logic is not scattered in page files

## Phase 4 Gate
Do not continue unless:
- member area feels like a real product
- library browsing works
- detail page is strong
- favorites behavior works
- account page exists
- mock data is structured

## Phase 5 Gate
Do not continue unless:
- admin pages feel functional
- create/edit flows work in demo form
- mock data services power the admin
- no hardcoded page-level asset chaos exists

## Phase 6 Gate
Do not continue unless:
- gated download states work
- blocked state is clear
- upgrade prompt exists
- billing page feels believable
- subscribed/unsubscribed states are easy to test

## Phase 7 Gate
Do not continue unless:
- app types are formalized
- repository/service boundaries are clean
- mocks are replaceable
- migration to real integrations looks realistic

## Phase 8 Gate
Do not continue unless:
- adapter boundaries exist
- future integration points are obvious
- no major rewrite appears necessary

## Phase 9 Gate
Do not continue unless:
- responsive behavior is good
- loading/empty states are polished
- visual consistency is strong
- nothing feels unfinished or cheap
